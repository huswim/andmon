# 시스템 아키텍처 및 최적화

이 문서는 Andmon 시스템의 고수준 아키텍처와 저지연, 고화질 유선 디스플레이 스트리밍을 구현하기 위해 적용된 주요 최적화 기법들을 설명합니다.

---

## 1. 시스템 아키텍처

Andmon은 Android Open Accessory (AOA) 프로토콜을 통신 기반으로 사용하는 전형적인 호스트-수신기(Host-Receiver) 모델을 따릅니다.

### 1.1 macOS 호스트 흐름
1.  **USB 탐색**: 연결된 Android 기기를 모니터링합니다.
2.  **AOA 핸드셰이크**: AOA 제어 요청(51, 52, 53)을 통해 태블릿을 USB 액세서리 모드로 전환합니다.
3.  **가상 디스플레이**: `VirtualDisplayBridge`를 사용하여 macOS 내에 가상 모니터를 생성합니다.
4.  **캡처**: **ScreenCaptureKit** (`SCStream`)을 사용하여 가상 디스플레이의 화면을 60 FPS로 캡처합니다.
5.  **인코딩**: 캡처된 프레임을 **VideoToolbox** (`VTCompressionSession`)로 전달하여 하드웨어 가속 **HEVC (H.265)**로 인코딩합니다.
6.  **전송**: 인코딩된 Annex B 데이터를 **Andmon Wire Protocol**로 캡슐화하여 USB bulk OUT으로 전송합니다.

### 1.2 Android 수신기 흐름
1.  **USB 액세서리**: OS가 액세서리 모드를 감지하고 Andmon 수신기 앱을 실행합니다.
2.  **프로토콜 파싱**: `FrameParser`를 사용하여 USB bulk IN 스트림으로부터 완전한 프레임을 재구성합니다.
3.  **디코딩**: H.265 비트스트림을 **MediaCodec**으로 전달하여 하드웨어 가속 디코딩을 수행합니다.
4.  **렌더링**: 디코딩된 프레임을 `SurfaceView`에 직접 렌더링하여 화면에 표시합니다.

---

## 2. 성능 최적화

Andmon의 최우선 목표는 50ms 미만의 저지연(Low Latency)입니다. 이를 위해 전체 스택에서 다양한 최적화가 적용되었습니다.

### 2.1 비디오 파이프라인 (지연 시간 및 품질)

*   **HEVC / H.265**: H.264보다 우수한 압축 효율을 제공하여 동일한 USB 대역폭에서 더 높은 화질을 유지합니다.
*   **B-프레임 제거**: `kVTCompressionPropertyKey_AllowFrameReordering`을 false로 설정합니다. 이를 통해 양방향 예측 프레임 생성에 필요한 대기 시간을 제거합니다.
*   **저지연 속도 제어 (Low Latency Rate Control)**: macOS 인코더에서 이 옵션을 활성화하여 USB 버스를 포화시키고 지터(jitter)를 유발할 수 있는 비트레이트 급증을 방지합니다.
*   **제조사 전용 디코더 힌트**: Android에서 `.low_latency` 하드웨어 디코더를 우선적으로 탐색하고, 제조사 전용 키(예: `vendor.qti-ext-dec-low-latency.enable`)를 적용하여 하드웨어 내부 버퍼링을 최소화합니다.
*   **실시간 스레드 우선순위**: Android의 데이터 읽기 및 디코딩 스레드를 `URGENT_DISPLAY` 및 `VIDEO` 우선순위로 설정하여 스케줄링 지터를 최소화합니다.

### 2.2 USB 전송 효율성

*   **LibUSB 통합**: macOS 호스트는 표준 OS 드라이버를 거치지 않고 `libusb`를 통해 직접 bulk 전송을 수행하여 오버헤드를 줄입니다.
*   **선택적 프레임 드랍**: USB 전송 큐가 쌓일 경우(예: 일시적인 버스 경합), 호스트는 대기 중인 P-프레임을 버리고 다음 IDR 프레임이나 키프레임 요청을 기다립니다. 이는 수신기가 실시간 스트림보다 뒤처지는 "지연 누적" 현상을 방지합니다.
*   **효율적인 MTU**: 프로토콜 수준에서 USB 분할/병합을 처리하여 헤더 오버헤드(24바이트)를 최소화하면서도 큰 페이로드(최대 8MiB)를 처리할 수 있습니다.

### 2.3 안정성 및 복구

*   **즉각적인 키프레임 요청**: 수신기에서 디코딩 오류나 데이터 손실을 감지하면 즉시 `KEYFRAME_REQUEST`를 보냅니다. 호스트는 즉시 IDR 프레임을 생성하여 스트림을 정상화합니다.
*   **시간 차이 방지 (Drift Prevention)**: 디코더 큐 진입부터 렌더링까지의 시간을 모니터링하여, 100ms 이상 지연된 프레임은 버림으로써 항상 최신 화면을 유지합니다.
*   **하트비트 (Ping/Pong)**: 2초 주기의 하트비트를 통해 케이블 단선이나 앱 종료를 감지합니다. 감지 시 호스트는 즉시 가상 디스플레이를 비활성화하고 인코더를 멈춰 자원을 보호합니다.

### 2.4 자원 관리

*   **Zero-Copy 메모리**: macOS에서 인코더 출력 데이터를 보낼 때 `memcpy`를 피하기 위해 메모리 포인터에 직접 접근하는 방식을 사용합니다.
*   **ByteBuffer 재사용**: Android의 `FrameParser`와 `HevcSurfaceDecoder`는 버퍼를 미리 할당하고 재사용하여 고대역폭 스트리밍 중의 GC(Garbage Collection) 부하를 줄입니다.
